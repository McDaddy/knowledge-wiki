## Overview

分布式训练之所以复杂，不只是因为模型大，更因为它把计算问题扩展成了通信问题。NCCL、RDMA、InfiniBand、RoCE、NVLink 和 NVSwitch 分别位于不同层级：NCCL 负责组织通信，RDMA 和 TCP 是传输方式，IB 与 RoCE 是网络实现，NVLink 与 NVSwitch 则负责单机内部 GPU 之间的高速互联。理解它们的分工，是判断训练瓶颈和排障路径的前提。

## Q1：为什么训练系统和推理系统应该分开理解？

训练和推理虽然都运行在 AI 基础设施上，但资源约束并不相同。训练更依赖计算能力、显存容量和跨卡跨机通信；推理更依赖时延、带宽和并发效率。

因此，训练系统通常必须同时考虑机内互联和机间通信，而推理系统很多时候只停留在单机多卡或小规模集群层面。把两者混在一起，会误判真正的性能瓶颈。

## Q2：NCCL、RDMA、IB、RoCE、NVLink 分别处在什么层？

最容易混淆的地方，是把这些名字都当作“同一种东西”。实际上它们位于完全不同的层级。

NCCL 是通信库，负责根据拓扑和设备状态组织集合通信；RDMA 是一种远程内存访问范式，强调绕过 CPU 参与数据传输；InfiniBand 是专用高性能网络体系；RoCE 是在以太网上实现 RDMA；NVLink 与 NVSwitch 则只解决单机内部 GPU 之间的高速互联问题。

一个更容易记忆的分层模型是：NCCL 在上层调度，RDMA/TCP 在传输层，IB/RoCE 在网络实现层，NVLink/NVSwitch 在机内互联层。

## Q3：AllReduce 在数据并行训练里到底做了什么？

数据并行的基本前提，是每张卡都保存一份相同模型，各自处理不同 batch 的数据，并分别计算梯度。AllReduce 的作用，是把这些卡上算出的梯度汇总并求平均，再让每张卡都用同一份结果更新参数。

这样做的意义，是让所有模型副本继续保持一致。也正因为如此，AllReduce 是强同步操作，只要有一张卡、一个节点或一段链路变慢，等待就会扩散到整个训练作业。

## Q4：为什么分布式训练容易出现“木桶效应”？

因为训练速度往往取决于最慢的那个环节，而不是平均水平。GPU 计算慢卡、网络丢包、链路降级、节点抖动，都会把等待传播给整个同步过程。

这意味着很多“训练慢”的问题，本质上不一定是 GPU 算不动，而是通信路径被最慢节点拖住。分布式训练越大，这种木桶效应越明显。

## Q5：为什么机内互联和机间网络要分开看？

单机内部的 GPU 互联，主要由 NVLink 和 NVSwitch 解决，它们决定同一台机器内部多卡协作时的数据交换效率。跨机器的训练通信，则更多依赖 InfiniBand、RoCE 以及 RDMA 能力。

这两类问题虽然都会影响训练速度，但排障思路不同。机内问题更接近硬件拓扑和板级互联问题，机间问题则更多涉及交换网络、链路质量、驱动栈和协议栈配置。

## Q6：理解通信栈，对训练排障有什么直接价值？

它能帮助区分“算力瓶颈”和“通信瓶颈”。如果只看到训练变慢，而没有把 NCCL、RDMA、IB、RoCE、NVLink 分层看清，就很容易把所有问题都归因于 GPU 本身。

一旦有了分层模型，排障顺序就更清晰了：先判断是机内还是机间，再判断是库层调度问题、传输方式问题、网络实现问题，还是底层链路与设备健康问题。对大规模训练来说，这种分层理解本身就是生产力。
